### Changes Overview
**Scope:** macOS permission app selection, release version sources, release notes

**Key Actions:**
- **[App Selection Hotfix]**: 当当前进程来自合法 `Open Computer Use.app` bundle 时，`currentAppBundleURL()` 优先返回正在执行的 bundle，避免 LaunchServices / Spotlight 返回仓库中残留的旧 staging app。
- **[Regression Test]**: 补充纯函数测试，锁定“运行中的 release bundle 可以覆盖 stale installed copy”的选择规则。
- **[Version Bump]**: 将插件 manifest、Swift/Go 运行时版本、smoke/test 初始化版本和 CLI helper 示例统一提升到 `0.1.38`。
- **[Release Notes]**: 记录 npm 安装后 `doctor` 启动错旧副本导致超时的修复。

### Design Intent (Why)
`0.1.37` 已把 macOS automation 转发给 app-agent，但 app 发现仍优先相信 `NSWorkspace.urlForApplication(withBundleIdentifier:)`。在开发机器上，LaunchServices 可能返回仓库里旧的 `dist/npm/.../Open Computer Use.app`，导致新 npm CLI 去启动旧 app-agent 并超时。当前进程所在的 `.app` 是用户实际执行的 npm 产物，应该优先作为 agent 启动目标。
